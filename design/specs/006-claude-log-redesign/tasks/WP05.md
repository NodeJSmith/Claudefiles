# WP05: Fix skill detection double-counting

## Scope

Deduplicate skill invocation counting in `cmd_skills` — prefer structured `Skill` tool_use, use XML tag as fallback, count each invocation once.

## Tasks

1. Refactor `cmd_skills` skill detection loop:
   - For each turn (user message + assistant response pair), collect skill names from both detection paths
   - XML tag path: `<command-name>/name</command-name>` in user messages
   - Tool_use path: `Skill` tool_use blocks in assistant messages
   - If same skill found via both paths in the same turn, count once, prefer tool_use source
2. Add `"via"` field to JSON output: `"tool_use"` or `"xml_tag"`
3. Add tests: turn with both signals produces one count, turn with only XML tag counted, turn with only tool_use counted
4. Run full test suite

## Acceptance criteria

- A skill invoked once per turn is counted once (not twice)
- `Skill` tool_use is preferred when both signals present
- JSON output includes `"via"` field indicating detection method
- All tests pass
