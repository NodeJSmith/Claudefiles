# WP05: Fix skill detection double-counting

## Scope

Deduplicate skill invocation counting in `cmd_skills` — a single skill invocation currently produces both an XML `<command-name>` tag in the user message AND a `Skill` tool_use block in the assistant response, causing double-counting. Fix by deduplicating per-turn.

## Files
- `bin/claude-log` — rewrite skill detection loop in `cmd_skills()` (L1082-1163) to deduplicate per turn; add `"via"` field to JSON output
- `tests/test_claude_log.py` — add tests for turn-based deduplication, `"via"` field, and single-signal detection

## Steps
1. Define "turn" for this context: a turn is the sequence of entries between consecutive `user`-type entries. Each `user` entry starts a new turn; the assistant entries and progress entries that follow belong to that turn until the next `user` entry.
2. Rewrite the `cmd_skills` detection loop:
   - Track current turn's skill detections in two sets: `xml_skills` and `tool_use_skills`
   - On each `user` entry: flush the previous turn's skills (merge, deduplicate, count), then reset both sets and scan for `<command-name>` tags → add to `xml_skills`
   - On each `assistant` entry: scan for `Skill` tool_use blocks → add to `tool_use_skills`
   - At end of file: flush the final turn
3. When merging a turn: for each unique skill name, count once. Record detection method: if in `tool_use_skills`, `via = "tool_use"`; if only in `xml_skills`, `via = "xml_tag"`
4. Add `"via"` field to JSON output (the most common detection method, or per-invocation if the output is per-invocation)
5. Add tests with JSONL fixtures:
   - Turn with both XML tag and Skill tool_use for same skill → counted once, `via: "tool_use"`
   - Turn with only XML tag → counted once, `via: "xml_tag"`
   - Turn with only Skill tool_use → counted once, `via: "tool_use"`
   - Two different skills in same turn → both counted
6. Run the test suite

## Verification

```
cd $REPO_ROOT && python -m pytest tests/test_claude_log.py -v -k "skill" 2>&1 | tail -30
```

## Done-when
- A skill invoked once per turn is counted once (not twice) even when both XML tag and tool_use are present
- `Skill` tool_use is the preferred detection method when both signals present
- JSON output includes `"via"` field indicating `"tool_use"` or `"xml_tag"`
- All tests pass

## Avoid
- Counting across turn boundaries — deduplication is per-turn only; the same skill invoked in two separate turns should count twice
- Breaking the normalization logic (colon → dot canonicalization at L1126-1134) — that stays
- Removing the `_BUILTIN_COMMANDS` filter — built-in commands should still be excluded
