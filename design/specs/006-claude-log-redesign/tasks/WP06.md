# WP06: Fix cmd_show duplication

## Scope

Eliminate the parallel `cmd_show` / `_filter_entries_for_show` code paths. After WP02 removes human-readable output, there's only one output path — unify into a single pass that builds a result list and emits JSON.

## Files
- `bin/claude-log` — delete `_filter_entries_for_show()` (L644-689); rewrite `cmd_show()` (L536-642) as a single-pass JSON builder reusing the existing agent index pattern
- `tests/test_claude_log.py` — convert `TestFilterEntriesForShow` (L965+) into tests for the unified `cmd_show`

## Steps
1. Delete `_filter_entries_for_show()` entirely (L644-689)
2. Rewrite `cmd_show()` as a single pass:
   - Iterate entries once via `iter_entries(path)`
   - Apply filters (messages, tools, user, assistant, thinking, usage) using the same logic currently in `_filter_entries_for_show`
   - Build agent index incrementally (same pattern as the current human-readable path at L587-593 and `iter_all_tool_calls` at L1277-1279)
   - For subagent tools: include `progress` entries when `include_subagents` is true, using `extract_subagent_tool_uses()` and the agent index for source labels
   - Collect results into a list, then `json.dumps` and print
3. Preserve the subagent tool output format from `_filter_entries_for_show`: `type`, `timestamp`, `name`, `input`, `source`, `agent_id`
4. Convert `TestFilterEntriesForShow` tests to exercise `cmd_show` directly (capture stdout JSON, parse, assert)
5. Run the test suite

## Avoid
- Building the agent index in a separate pre-pass — keep it incremental (Agent tool_use entries always precede their progress entries in the log)
- Changing the output schema for subagent tools — existing callers may depend on `type`, `name`, `input`, `source`, `agent_id` fields
- Removing the `show_all` logic — when no filter flags are set, all entry types should be included
