# WP03: Fix path decoding and optimize find_sessions

## Scope

Fix the lossy path decoding bug and optimize `find_sessions` to read only the first ~20 lines per file instead of every line.

## Tasks

1. Add `extract_cwd(path)` function: scan first 20 entries of type `user` or `assistant` for a `cwd` field. Return the cwd string or `None`.
2. Update `find_sessions` to use `extract_cwd` instead of `project_path_from_dir` for the `project_path` field
3. If `extract_cwd` returns `None`, fall back to `project_path_from_dir` (lossy but better than nothing)
4. Update `--cwd` filter in `iter_session_files` to match against the real `cwd` value
5. Optimize `find_sessions`: read only first ~20 lines per file for metadata (timestamp, branch, cwd). Estimate `msg_count` from line count (count newlines without full JSON parse).
6. Keep `project_name_from_dir` as-is (last segment of encoded dirname is fine for display)
7. Keep `project_path_from_dir` as fallback only
8. Add tests: `extract_cwd` with various entry types, `find_sessions` optimization correctness
9. Update `TestProjectPathFromDir` — the "known limitation" test can note the fallback path

## Acceptance criteria

- `list --json` shows correct `project_path` for hyphenated projects (when cwd is in JSONL)
- `--cwd my-project` matches sessions with `/home/user/my-project` as working directory
- `list` reads ~20 lines per file, not the entire file
- All tests pass
