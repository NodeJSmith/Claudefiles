# WP03: Fix path decoding and optimize find_sessions

## Scope

Fix the lossy path decoding bug (hyphenated project names silently corrupted) and optimize `find_sessions` to read only the first ~20 lines per file instead of every line.

## Files
- `bin/claude-log` — add `extract_cwd()` function; rewrite `find_sessions()` (L137-186) to use `extract_cwd` and estimate `msg_count` from line count; update `iter_session_files()` `--cwd` filter (L77-115) to match against real cwd
- `tests/test_claude_log.py` — add tests for `extract_cwd`; update `TestProjectPathFromDir` (L55-76) to note fallback-only role; add tests for `find_sessions` optimization correctness

## Steps
1. Add `extract_cwd(path: Path) -> str | None`: open file, read up to 20 lines, parse each as JSON, return the first `cwd` field found on entries of type `user` or `assistant`, or `None`
2. Update `find_sessions()` to call `extract_cwd()` for the `project_path` field instead of `project_path_from_dir()`; fall back to `project_path_from_dir()` when `extract_cwd` returns `None`
3. Optimize `find_sessions()`: stop parsing every line for `msg_count`; instead count total newlines in the file (e.g., `sum(1 for _ in f)`) without JSON-parsing, and extract metadata (`timestamp`, `gitBranch`, `cwd`) from only the first ~20 lines
4. Update `iter_session_files()` `cwd_filter` matching to use `extract_cwd()` against the real cwd value (currently it's unused or uses the dirname)
5. Keep `project_name_from_dir` unchanged — it's fine for display (last path segment)
6. Keep `project_path_from_dir` as fallback only — add a comment noting it's lossy
7. Add tests: `extract_cwd` with JSONL containing cwd in first entry, cwd in 15th entry, no cwd at all; `find_sessions` returns correct `project_path` for hyphenated names
8. Run the test suite

## Done-when
- `list` output shows correct `project_path` for projects with hyphens in their names (when cwd is in the JSONL)
- `--cwd my-project` matches sessions whose real working directory contains `my-project`
- `find_sessions` reads at most ~20 lines of JSON per file (the rest is newline counting only)
- `project_path_from_dir` is still present but only used as a fallback with a comment
- All tests pass

## Avoid
- Removing `project_path_from_dir` — it's the fallback for old sessions without `cwd` in JSONL
- Reading the entire file to count messages — use newline counting (`sum(1 for _ in f)`) without JSON parsing
- Breaking `resolve_session` — it doesn't use `project_path` so no changes needed there
