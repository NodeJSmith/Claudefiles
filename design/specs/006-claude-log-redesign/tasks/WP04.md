# WP04: Fix search/grep regex consistency

## Scope

Align `search` to `grep` — both regex by default, with `--fixed` / `-F` for literal matching.

## Pre-work

- [ ] Grep repo for `claude-log search` to check if any callers rely on literal matching

## Tasks

1. `cmd_search`: remove `re.escape()` wrapper — query is now compiled as regex directly
2. Both `cmd_search` and `cmd_grep` (via `iter_all_tool_calls`): wrap `re.compile` in try/except, print user-friendly error to stderr on `re.error`
3. Add `--fixed` / `-F` flag to both `search` and `grep` argparse — when set, applies `re.escape()` before compilation
4. Update help text to document regex behavior
5. Add tests: regex patterns in search, `--fixed` flag, bad regex error handling

## Acceptance criteria

- `claude-log search "error.*timeout"` works (regex)
- `claude-log search -F "error.*timeout"` treats pattern as literal
- `claude-log grep "git (push|pull)"` still works (regex, unchanged)
- Invalid regex produces a clear error message, not a stack trace
- All tests pass
