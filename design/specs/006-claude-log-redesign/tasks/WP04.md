# WP04: Fix search/grep regex consistency

## Scope

Align `search` to `grep` ŌĆö both regex by default, with `--fixed` / `-F` for literal matching. The `mine.tool-gaps` skill passes literal strings like `"| jq"` and `" & "` to `claude-log search` that contain regex metacharacters ŌĆö these callers must be updated to use `-F`.

## Files
- `bin/claude-log` ŌĆö remove `re.escape()` in `cmd_search` (L755); add `--fixed`/`-F` flag to `search` and `grep` argparse; wrap `re.compile` in try/except for both commands
- `tests/test_claude_log.py` ŌĆö add tests for regex search, `--fixed` flag, bad regex error handling
- `skills/mine.tool-gaps/SKILL.md` ŌĆö update `claude-log search` calls (L83: `"| jq"`, L89: `" & "`) to use `-F` flag for literal matching

## Steps
1. In `cmd_search` (L755), remove the `re.escape()` wrapper: change `re.compile(re.escape(args.query), re.IGNORECASE)` ŌåÆ `re.compile(args.query, re.IGNORECASE)` (or `re.compile(re.escape(args.query), ...)` when `args.fixed` is set)
2. Add `--fixed` / `-F` flag to `search` argparse (after L1611): when set, applies `re.escape()` before `re.compile`
3. Add `--fixed` / `-F` flag to `grep` argparse (after L1687): same behavior
4. In both `cmd_search` and `cmd_grep` (via `iter_all_tool_calls`), wrap `re.compile` in try/except `re.error` ŌĆö print a user-friendly message to stderr and `sys.exit(1)`
5. Update `skills/mine.tool-gaps/SKILL.md` search commands to use `-F`:
   - L83: `claude-log search "| jq"` ŌåÆ `claude-log search -F "| jq"`
   - L89: `claude-log search " & "` ŌåÆ `claude-log search -F " & "`
   - L80: `claude-log search "python3 -c"` ŌĆö already literal-safe (no metacharacters), but add `-F` for consistency
   - L86: `claude-log search "curl"` ŌĆö literal-safe, add `-F` for consistency
6. Update help text for `search` and `grep` to document regex-by-default behavior
7. Add tests: regex pattern in search works, `--fixed` treats metacharacters literally, invalid regex gives clean error (not traceback)
8. Run the test suite
