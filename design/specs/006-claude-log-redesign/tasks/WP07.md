# WP07: JSONL corruption warnings and --limit

## Scope

Minor improvements: warn on corrupt JSONL lines, add `--limit` to `show` and `extract`.

## Tasks

1. `iter_entries`: count skipped lines per file. If >1 line skipped, emit warning to stderr: `"Warning: skipped N corrupt lines in {path}"`
2. Add `--limit` / `-l` to `show` and `extract` argparse (default: unlimited)
3. Wire `--limit` through `cmd_show` and `cmd_extract` to cap output entries
4. Add tests: corrupt JSONL produces warning, `--limit` caps output
5. Run full test suite

## Acceptance criteria

- Corrupt JSONL files produce a stderr warning (not silent)
- A single corrupt last line (common for active sessions) does NOT warn
- `show --limit 10` returns at most 10 entries
- `extract --tools --limit 50` returns at most 50 tool calls
- All tests pass
