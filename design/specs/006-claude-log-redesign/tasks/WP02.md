# WP02: Collapse output to JSON-only

## Scope

Remove the ANSI color system, table formatter, and all human-readable output paths. Keep only JSON output. ~240 lines removed.

## Pre-work

- [ ] Audit uses of `truncate()` and `_summarize_tool_input()` — determine if they're used in JSON result building (not just human display)
- [ ] Grep repo for `claude-log.*--json`, `claude-log.*--no-color`, `claude-log.*--no-subagents`

## Tasks

1. Remove ANSI constants: `BOLD`, `DIM`, `RED`, `GREEN`, `YELLOW`, `BLUE`, `MAGENTA`, `CYAN`, `RESET`
2. Remove `USE_COLOR` global, `c()` helper, `--no-color` flag handling in `_add_global_opts`
3. Remove `format_table()`
4. Remove `truncate()` — replace any JSON-path usage with inline slice or remove
5. Audit `_summarize_tool_input()` — if only used for human display, remove. If used in JSON results (e.g., `cmd_search`), keep or replace with raw tool input
6. In each `cmd_*` handler, remove the human-readable `else` branch — keep only the JSON path
7. Add backward-compat no-ops: `--json`, `--no-color`, `--no-subagents` accepted silently
8. Remove `--json` from `_add_global_opts` (or make it a silent no-op there)
9. Update tests: remove `TestFormatTable`, `TestColorWrapper`, `TestTruncate`, `TestSummarizeToolInput` (if function removed)
10. Run full test suite

## Acceptance criteria

- All commands emit JSON by default (no `--json` flag needed)
- `--json`, `--no-color`, `--no-subagents` accepted without error (silent no-ops)
- No ANSI escape codes in any output
- All tests pass
- ~240 lines removed
