# WP02: Collapse output to JSON-only

## Scope

Remove the ANSI color system, table formatter, and all human-readable output branches (~240 lines). Keep only JSON output paths. `_summarize_tool_input()` stays — it IS used in JSON search results via `_match_entry` (L748) and `cmd_search` (L791, L798).

## Files
- `bin/claude-log` — remove color constants, `c()`, `USE_COLOR`, `format_table()`, `truncate()`, human-readable branches in every `cmd_*`; rename `_summarize_tool_input` to `_tool_input_summary` to clarify it's for search result text; replace `truncate()` calls in JSON paths with inline slices
- `tests/test_claude_log.py` — remove `TestFormatTable` (L483-514), `TestColorWrapper` (L517-538), `TestTruncate` (L303-330); update `TestSummarizeToolInput` class name to match rename

## Steps
1. Remove ANSI constants: `BOLD`, `DIM`, `RED`, `GREEN`, `YELLOW`, `BLUE`, `MAGENTA`, `CYAN`, `RESET` (L34-42), `USE_COLOR` (L44), `c()` (L47-51)
2. Remove `format_table()` (L309-326)
3. Remove `truncate()` (L328-338) — replace its 3 JSON-path usages: `cmd_search` L778 `truncate(matched_text, 120)` → `matched_text[:120]`, L798 `truncate(f"[{name}] {summary}", 120)` → `f"[{name}] {summary}"[:120]`, `_summarize_tool_input` L714 `truncate(str(v), 40)` → `str(v)[:40]`
4. Rename `_summarize_tool_input` → `_tool_input_summary` (clarifies it's for search result text, not just human display)
5. In each `cmd_*` handler, remove the human-readable `else` branch — keep only the JSON path; make JSON the default (no `if args.json:` guard)
6. Remove `global USE_COLOR` from `main()` and the `args.no_color` handling
7. In `_add_global_opts`, make `--json` and `--no-color` silent no-ops (accept but ignore). Add `--no-subagents` as a silent no-op at the global level too
8. Remove test classes: `TestFormatTable`, `TestColorWrapper`, `TestTruncate`
9. Rename `TestSummarizeToolInput` → `TestToolInputSummary`
10. Run the test suite
