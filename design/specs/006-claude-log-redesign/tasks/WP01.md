# WP01: Delete permissions subsystem

## Scope

Remove the self-contained permissions subsystem (~160 lines of functions + ~15 lines of argparse + tests) and update the `mine.permissions-audit` command that calls `claude-log permissions`.

## Files
- `bin/claude-log` — remove `SETTINGS_PATH`, `load_allowed_patterns()`, `tool_signature()`, `is_allowed()`, `_find_project_root()`, `suggest_pattern()`, `cmd_permissions()`; replace with deprecation stub in argparse
- `tests/test_claude_log.py` — remove `TestToolSignature`, `TestIsAllowed`, `TestSuggestPattern`
- `commands/mine.permissions-audit.md` — update to stop calling `claude-log permissions` (line 11: `$ARGUMENTS` passthrough, line 20: `claude-log permissions --json`, line 27: references its output)

## Steps
1. Read `commands/mine.permissions-audit.md` fully and decide replacement strategy (the command already reads debug logs in Phase 2 — that path stays; remove the `claude-log permissions` Phase 1)
2. Remove functions: `load_allowed_patterns()` (L343-357), `tool_signature()` (L359-389), `is_allowed()` (L391-419), `_find_project_root()` (L421-462), `suggest_pattern()` (L464-502), `cmd_permissions()` (L1430-1531), `SETTINGS_PATH` (L340)
3. In `build_parser()`, replace the `permissions` subparser (L1704-1717) with a stub that prints a deprecation message to stderr and exits 0
4. Remove test classes: `TestToolSignature` (L334-401), `TestIsAllowed` (L404-446), `TestSuggestPattern` (L449-479)
5. Update `commands/mine.permissions-audit.md` to remove the `claude-log permissions` call and adjust the skill to use only the debug-log-based analysis
6. Run the test suite
