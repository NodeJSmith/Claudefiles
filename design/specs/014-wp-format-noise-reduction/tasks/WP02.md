## Objectives & Success Criteria

- `spec-helper design-extract <feature>` outputs extracted sections from design.md to stdout
- Supports `--sections` flag to specify which sections to extract (default: `Architecture "Non-Goals"`)
- Supports heading aliases: `## Architecture`, `## Proposed Approach`, `## Technical Approach` all match the Architecture slot
- Exits non-zero with clear error when no canonical heading is found
- Handles missing `## Non-Goals` gracefully (extracts Architecture only + warning to stderr)
- Includes nested `###` subsections under matched `##` headings
- `test_design_extract_*` tests all pass

## Subtasks

1. In `packages/spec-helper/src/spec_helper/filesystem.py`: add `extract_design_sections(feature_dir: Path, sections: list[str]) -> str` function. For each requested section name, find the `## <name>` heading (with alias matching), extract all content until the next `## ` heading or EOF (including nested `###` subsections). Return the concatenated extracted sections as a string. Alias map: `{"Architecture": ["Proposed Approach", "Technical Approach"]}`
2. In `packages/spec-helper/src/spec_helper/commands.py`: add `cmd_design_extract(args)` function. Reads `args.feature` to resolve feature dir, calls `extract_design_sections()` with the requested sections, prints result to stdout. Exits non-zero if the primary section (Architecture) is not found.
3. In `packages/spec-helper/src/spec_helper/cli.py`: add `design-extract` subparser with `feature` positional arg and `--sections` nargs='+' argument (default `["Architecture", "Non-Goals"]`). Wire to `cmd_design_extract` in the dispatch dict.
4. Add `--reviewer` convenience flag to `design-extract` subparser that sets sections to `["Architecture", "Non-Goals", "Alternatives Considered"]` — this is the spec reviewer extract preset
5. Write tests for `extract_design_sections()` and `cmd_design_extract()`

## Test Strategy

Tests in `tests/test_spec_helper.py`:

- `test_design_extract_architecture` — design.md with `## Architecture` heading extracts correctly including `###` subsections
- `test_design_extract_proposed_approach_alias` — design.md using `## Proposed Approach` matches the Architecture alias
- `test_design_extract_non_goals` — extracts `## Non-Goals` content
- `test_design_extract_missing_architecture_exits_nonzero` — design.md with no Architecture/Proposed Approach heading causes non-zero exit
- `test_design_extract_missing_non_goals_warns` — design.md with Architecture but no Non-Goals produces Architecture-only output + stderr warning
- `test_design_extract_reviewer_flag` — `--reviewer` flag adds Alternatives Considered to the extract
- `test_design_extract_nested_subsections` — `### ` headings under `## Architecture` are included in the extract

## Review Guidance

- **FAIL if**: extraction includes content from sections NOT requested (e.g., `## Problem` leaks into output)
- **FAIL if**: `###` subsections under a `##` heading are excluded from the extract
- **WARN if**: alias matching is case-sensitive (should be case-insensitive for robustness)
- The function must handle design docs with varying heading styles — some use `## Architecture`, some `## Proposed Approach`. The alias map is the single source of truth for equivalences.
- `--reviewer` is a convenience shortcut, not a separate code path — it just sets `--sections` to a preset list
