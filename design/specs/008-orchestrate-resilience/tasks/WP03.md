## Objectives & Success Criteria

When the spec reviewer returns WARN, the orchestrator automatically re-runs the executor with the spec reviewer's feedback, then re-runs the spec reviewer. Max 1 automatic retry. On second WARN, escalate to the user. The "Previous review feedback" section is added to the executor prompt for both WARN and FAIL retries.

Success:
- WARN at Step 5 triggers one automatic retry: executor re-run with feedback → spec reviewer re-run
- After retry, if PASS → continue to Step 7 (code reviewer) as normal
- After retry, if still WARN → escalate to user (FAIL gate options)
- The executor prompt includes a `## Previous review feedback` section on retry with spec reviewer details
- For FAIL retries (after Step 9), the feedback section includes code reviewer and visual reviewer details too
- Only the most recent attempt's feedback is included (not accumulated)
- Feedback is truncated to 50 lines if it exceeds that length
- The WARN retry replaces only Steps 4 and 5; Steps 7, 8, and 8.5 still run
