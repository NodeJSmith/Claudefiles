## Objectives & Success Criteria

The 19 i-* skills live in `skills-impeccable/`, the 3 cm-* skills live in `skills-memory/`, and `rules/common/capabilities.md` is split into three conditional fragments. All relative path references between i-* skills remain valid. Running `install.sh` (the current installer) still works — symlinks are created from both new directories into `~/.claude/skills/`.

## Subtasks

1. `git mv` all 19 `skills/i-*` directories to `skills-impeccable/` (including `i-frontend-design` with its `reference/` subdirectory)
2. `git mv` all 3 `skills/cm-*` directories to `skills-memory/`
3. Update `install.sh` temporarily to iterate `skills-impeccable/` and `skills-memory/` in addition to `skills/` — this keeps the current installer functional until WP03 replaces it
4. Split `rules/common/capabilities.md` into three files:
   - `rules/common/capabilities-core.md` — the header/preamble, the `## Intent Routing` table (mine.* rows only), and the `## CLI Tools` section
   - `rules/common/capabilities-impeccable.md` — the 19 i-* trigger rows as a standalone table with the same header format
   - `rules/common/capabilities-memory.md` — the 3 cm-* trigger rows as a standalone table with the same header format
5. Delete the original `rules/common/capabilities.md` after the split
6. Verify all `../i-frontend-design/reference/` relative paths in the 10 i-* SKILL.md files resolve correctly from `skills-impeccable/` (they should, since all i-* skills move together)

## Test Strategy

N/A — no testable code changes. Directory moves and file splits are verified by running `install.sh` and confirming symlinks are created correctly. Verify relative paths by reading the referenced files from the new locations.

## Review Guidance

- Confirm all 19 i-* skills are in `skills-impeccable/` and none remain in `skills/`
- Confirm all 3 cm-* skills are in `skills-memory/` and none remain in `skills/`
- Confirm `capabilities-core.md` does NOT contain any i-* or cm-* trigger rows
- Confirm `capabilities-impeccable.md` contains exactly the 19 i-* rows and no mine.* rows
- Confirm `capabilities-memory.md` contains exactly the 3 cm-* rows and no mine.* rows
- Confirm the original `capabilities.md` is deleted
- Confirm `install.sh` iterates the new directories so existing users are not broken during the transition
