## Objectives & Success Criteria

All hook entries in `settings.json` that reference symlinked scripts use self-guarding wrappers, so a missing script exits 0 (passthrough) instead of exit 127 (denial). All agent `.md` files have a `group:` field in their YAML frontmatter. These are prerequisites for the installer's hook deselection and agent grouping features.

## Subtasks

1. Update `settings.json` PreToolUse hook entries for `sudo-poll.sh`, `pytest-guard.sh`, and `pytest-loop-detector.sh` — wrap each in `bash -c '[ -x "${CLAUDE_HOME:-$HOME/.claude}/scripts/hooks/<script>" ] && exec "${CLAUDE_HOME:-$HOME/.claude}/scripts/hooks/<script>" || exit 0'` pattern, matching the existing cm-* hook style at lines 122-160
2. Update `settings.json` PostToolUse hook entries for `pytest-loop-reset.sh` and `pytest-loop-status.sh` with the same self-guarding wrapper pattern
3. Add `group: core` to frontmatter of: `architect.md`, `code-reviewer.md`, `integration-reviewer.md`, `issue-refiner.md`, `planner.md`, `qa-specialist.md`, `researcher.md`, `visual-diff.md`
4. Add `group: engineering` to frontmatter of: `engineering-backend-developer.md`, `engineering-data-engineer.md`, `engineering-frontend-developer.md`, `engineering-sre.md`, `engineering-technical-writer.md`, `testing-reality-checker.md`
5. Add `group: memory` to frontmatter of: `cm-memory-auditor.md`, `cm-signal-discoverer.md`
6. Run `claude-merge-settings` to propagate the settings.json changes

## Test Strategy

N/A — no testable code changes. Settings.json and agent frontmatter are configuration files. Verification is manual: confirm `settings.json` hooks fail gracefully when a symlink is removed, and confirm agent frontmatter parses correctly.
