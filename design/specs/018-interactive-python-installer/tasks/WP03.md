## Objectives & Success Criteria

A single `install.py` script with PEP 723 inline metadata exists at the repo root. Running `uv run install.py` presents a guided wizard on first run, saves selections to `${CLAUDE_HOME}/.claudefiles-install-config.json`, and creates symlinks + installs packages per the user's choices. Re-runs are silent unless new groups appear. `--reconfigure` shows the full wizard. `--uninstall` cleans up. Non-interactive mode works without prompts. Tests pass for core logic.

## Subtasks

1. Create `install.py` with PEP 723 inline metadata header (`rich>=13.0`, `questionary>=2.0`, `requires-python = ">=3.10"`)
2. Implement CLI argument parsing: `--reconfigure`, `--uninstall`, no args (default)
3. Implement config loading from `${CLAUDE_HOME:-~/.claude}/.claudefiles-install-config.json` with JSON schema version check and corrupt-file recovery (treat as first run)
4. Implement config saving with atomic writes (`tempfile` + `os.replace()`) — write intent before installation, update after completion
5. Implement concurrency guard using `fcntl.flock` on the config file
6. Define asset group data structures: `SKILL_GROUPS` (3 groups mapping to `skills/`, `skills-impeccable/`, `skills-memory/`), `HOOK_GROUPS` (3 groups: pytest with 6 files, sudo with 1, tmux with 1), `PACKAGE_DEFS` (4 packages with dependency auto-selection rules)
7. Implement agent group discovery by reading `group:` frontmatter from each `.md` file in `agents/`
8. Implement the wizard flow using `questionary.checkbox()`: skill groups → agent groups → hook groups → packages (with auto-selection logic for claude-memory when cm-* skills/agents are selected)
9. Implement smart diff: compare saved config keys against current group definitions, prompt only for new groups not in config
10. Implement symlink operations for all 5 categories: directory-level (skills, agents, commands, hooks), file-level (rules, learned, bin)
11. Implement ownership guard using `Path.is_relative_to(repo_dir)` for deselection cleanup
12. Implement stale symlink detection using `os.readlink()` (raw) for ownership checking on broken symlinks
13. Implement shadowed file detection and interactive replacement prompt
14. Implement `uv tool install -e` execution for selected packages with error handling (continue on failure, report errors)
15. Implement `--uninstall`: remove Claudefiles-owned symlinks, run `uv tool uninstall` for packages in config, delete config file
16. Implement non-interactive fallback: detect `not sys.stdin.isatty()`, use saved config or default to all groups, do NOT save config in this mode
17. Wrap `questionary` import in try/except for `prompt-toolkit` terminal detection errors, falling back to non-interactive mode with warning
18. Write unit tests in `tests/test_install.py`:
    - `test_config_load_save_roundtrip` — write config, read back, verify contents
    - `test_config_corrupt_recovery` — write invalid JSON, verify installer treats as first run
    - `test_config_atomic_write` — verify tempfile + os.replace pattern
    - `test_ownership_guard_relative_to` — create symlinks to repo dir and outside, verify only repo symlinks are identified as owned
    - `test_ownership_guard_prefix_collision` — create `Claudefiles-backup/` sibling, verify it's NOT treated as owned
    - `test_stale_symlink_detection` — create broken symlink, verify it's detected and ownership is checked via readlink
    - `test_smart_diff_new_group` — saved config missing a group key, verify it's flagged as new
    - `test_smart_diff_no_changes` — saved config matches current groups, verify no prompt needed
    - `test_symlink_creation` — verify directory-level and file-level symlinks are created correctly in a tmp_path
    - `test_deselection_cleanup` — verify deselected group symlinks are removed, non-owned symlinks are preserved
19. Write `test_questionary_import_fallback` — verify the try/except wrapper around questionary import falls back to non-interactive mode (test by monkeypatching the import to raise)
20. Write integration test `test_full_install_flow` — create a mock repo structure in tmp_path, run the installer's core logic (not the wizard UI), verify correct symlinks exist

## Test Strategy

Tests in `tests/test_install.py` using pytest with `tmp_path` fixtures. Each test creates a minimal directory structure simulating the Claudefiles repo layout and a target `~/.claude/` equivalent. The ownership guard tests create sibling directories to verify `is_relative_to()` correctness. Integration test runs the full symlink pipeline against a mock repo. Wizard UI is tested manually.

Test functions: `test_config_load_save_roundtrip`, `test_config_corrupt_recovery`, `test_config_atomic_write`, `test_ownership_guard_relative_to`, `test_ownership_guard_prefix_collision`, `test_stale_symlink_detection`, `test_smart_diff_new_group`, `test_smart_diff_no_changes`, `test_symlink_creation`, `test_deselection_cleanup`, `test_questionary_import_fallback`, `test_full_install_flow`.

## Review Guidance

- Verify `Path.is_relative_to()` is used everywhere ownership is checked — no `startswith` anywhere
- Verify stale symlink detection uses `os.readlink()` (raw), not `Path.resolve()`
- Verify non-interactive mode does NOT save a config file when no prior config exists
- Verify auto-selection logic: selecting cm-* skills/agents auto-selects claude-memory package
- Verify `--uninstall` runs `uv tool uninstall` for each package listed in the config
- Verify the atomic config write pattern uses `tempfile` + `os.replace()`, not direct write
- Verify `fcntl.flock` is used for concurrency protection
- Verify all tests pass and cover the critical ownership guard logic
