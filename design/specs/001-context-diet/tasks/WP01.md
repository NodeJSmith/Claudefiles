# WP01: Write promptfoo routing eval tests

## Objectives & Success Criteria

- [ ] Create `evals/compliance/routing/intent-to-skill.yaml` with 10-15 representative skill routing test cases
- [ ] Create `evals/compliance/routing/intent-to-agent.yaml` with 5-8 agent routing test cases

- [ ] Baseline pass rate recorded (does not need to be 100%)

## Subtasks

1. Review existing eval structure (`evals/compliance/tools/gh-pr-reply.yaml`, `evals/lib/assert-tool-called.js`) for patterns
2. Write skill routing tests — cover key intents from the routing table: ship, commit-push, brainstorm, research, refactor, audit, build, design, specify, security-review
3. Write agent routing tests — cover: code-reviewer (auto-trigger on code changes), planner (complex features), qa-specialist
4. Write CLI tool routing tests — cover: gh-pr-reply (already has a test, extend or reference), claude-tmux, claude-log
5. Run baseline: `npx promptfoo eval -c evals/compliance/routing/` and record results
6. Document baseline pass rate in this WP's Activity Log

## Test Strategy

Tests use `anthropic:claude-agent-sdk` provider with real Claude Code sessions. Assertions check that the correct tool/skill/agent is invoked via `toolCalls` metadata. Use the existing `assert-tool-called.js` helper library.

**Key assertion patterns:**
- Skill invocation: check for `Skill` tool call with matching skill name in input
- Agent invocation: check for `Agent` tool call with matching `subagent_type` in input
- CLI tool invocation: check for `Bash` tool call containing the tool name in command

## Review Guidance

- Tests should be representative, not exhaustive — 20-28 total test cases across the three files
- Focus on the most commonly used routes, not edge cases
- Each test case should have a clear, natural-language prompt (the way a user would actually say it)
