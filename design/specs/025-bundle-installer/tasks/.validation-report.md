## Warnings

1. **T03 has no AC coverage in the design doc.** T03's Focus section notes this explicitly: "There is no AC in the design doc for FR#13 (it was added after the AC list was written)." FR#13 is covered in the Verify section, but there is no AC#N in the design doc for the ADO detection scenario. This is a design doc incompleteness, not a task file deficiency — the task correctly anchors verification directly to FR#13.

2. **T01 Verify item for FR#10 is weaker than the AC.** The T01 Verify item for FR#10 states "`capabilities-core.md` is not listed in any bundle and installs via the always-installed `rules/` symlinks." AC#13 states "`capabilities-core.md` present in `rules/common/` regardless of bundle selections." The Verify item is structural (not in any bundle) rather than behavioral (present after install). Both the AC and the Verify item appear in T01, so the behavioral check is also present — but a reader relying solely on the FR#10 Verify item would not catch a regression where the file fails to install.

3. **T04 depends on T01 but T01's BUNDLES may differ from the design draft.** T04's Focus explicitly flags: "If T01's `BUNDLES` differs from the design in any detail, the docs follow the code, not the design draft." This is correct guidance but means the ONBOARDING.md content cannot be fully validated against the design doc in isolation — its accuracy is contingent on T01's implementation.
