## Objectives & Success Criteria

Insert the three-stage pre-flight analysis sub-phase into SKILL.md Phase 1, between "Gather context" (line 196) and "Specialist Selection" (line 217). After this WP, running `/mine.challenge` on any target will: (1) scan for surface issues using target-type-aware heuristics, (2) present found issues via AskUserQuestion with Fix all / Show me each fix / Skip options, (3) run an architecture smell test that gates critic launch, and (4) generate a critic briefing block.

## Subtasks

1. Add a new `### Pre-Flight Analysis` heading after the "Gather context" section (after line 196, before `### If empty`) in `skills/mine.challenge/SKILL.md`. Insert the three stages as sub-sections.

2. Write Stage 1: Surface Issue Scan. Include the target-type-aware heuristics table from the design doc (design-doc/spec, skill-file, code, frontend-code, brief/research, rule, agent-file/docs/other). Add the AskUserQuestion template with three options: "Fix all", "Show me each fix", "Skip fixes". Specify the "Show me each fix" interaction pattern: sequential AskUserQuestion calls per fix with Accept/Skip options.

3. Write Stage 2: Architecture Smell Test. Include the guiding question ("Would critics all point to the same root cause?"), the four check items, the over-sensitivity instruction, and the AskUserQuestion gate with "Proceed anyway" / "Let me rethink" options. Specify that when no concerns are detected, proceed silently.

4. Write Stage 3: Critic Briefing Generation. Include the briefing template with Problem Context block scoped to design-doc/spec targets only (omitted for all other types). Include the Review Focus section (applies to all targets). Include two conditional sections: "Known Surface Issues" for user-skipped issues ("deprioritize — user reviewed and declined"), and "Pre-Flight Notes" for inline-content unfixable issues ("weigh normally").

5. Add edge case handling inline with the pre-flight instructions: (a) no surface issues found → skip AskUserQuestion, proceed silently; (b) inline content targets → cannot edit, inject as briefing notes using the "Pre-Flight Notes" template; (c) user skips all fixes → note in briefing as "Known Surface Issues".
