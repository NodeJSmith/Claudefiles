## Objectives & Success Criteria

Add re-challenge detection logic to Phase 1 and restructure the manifest write to happen in two steps (Phase 1 metadata + Phase 2 critic list). After this WP: (1) re-challenge is detected via on-disk artifact check as the primary signal, (2) the `# rechallenge: yes | no` field is written to the manifest at the end of Phase 1, and (3) specialist selection is skipped on detected re-challenges.

## Subtasks

1. Add a new `### Re-Challenge Detection` section in `skills/mine.challenge/SKILL.md` Phase 1, after the Pre-Flight Analysis sub-phase (added in WP01) and before Specialist Selection. The detection logic: check whether a prior findings file exists at the `--findings-out` path (if provided) or in the same feature directory. If a prior findings file exists, mark as re-challenge. Conversation context is a secondary signal only.

2. Modify the Specialist Selection section to check the re-challenge flag. If re-challenge is detected, skip specialist selection entirely and note "Specialists skipped (re-challenge)."

3. Move the manifest write from Phase 2 (`### Write session manifest`, currently at SKILL.md line 329) to Phase 1. Add it as a new `### Write session manifest` section at the end of Phase 1, after Specialist Selection. The Phase 1 manifest write includes all comment-line metadata (`# target-type`, `# mode`, `# findings-out`, `# focus`, `# target`, and the new `# rechallenge: yes | no`). The critic file list is left empty — Phase 2 appends it after roster finalization.

4. Update the Phase 2 manifest section (previously "Write session manifest") to become "Update manifest with critic file list" — it reads the existing manifest from Phase 1 and appends the critic report filenames (generics + any specialists). The comment-line metadata is already present from Phase 1.

5. Update the `### Validate critic reports` section to note that on re-challenges, the expected critic count is 2 (not 3-5), so the confidence denominator adjusts accordingly.

## Test Strategy

N/A — no testable code changes. Validation is manual: run `/mine.challenge` twice on the same target in one session and verify the second run detects the prior findings file and marks `# rechallenge: yes` in the manifest.

- The existing manifest format docs (field definitions, mode/findings-out/focus/target) must not be duplicated — keep them in one place and reference from the other
