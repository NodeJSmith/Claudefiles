## Contradictions

- **T05 Phase 5 (Review) Prompt**: `"cfl gate plan-review --verdict <PASS|WARN|FAIL>. Use the reviewer's verdict directly."` conflicts with the actual reviewer subagent's overall verdict enum defined in `skills/mine-plan/SKILL.md:386` (`"Verdict — PASS, FAIL, or ABANDON"`) and `skills/mine-plan/reviewer-prompt.md:57,70-73` (`"## Verdict: PASS | FAIL | ABANDON"`). The reviewer's overall verdict is never `WARN` (WARN only applies to individual checklist line items, not the overall verdict), and it can be `ABANDON`, which is not a member of cfl's `COMMON_VERDICTS` (`PASS, WARN, FAIL, SKIPPED` — `packages/cfl/src/cfl/vocabulary.py:9`). If T05 is implemented as written and the reviewer returns `ABANDON`, `cfl gate plan-review --verdict ABANDON` will fail with `invalid_verdict` (exit code 2, per `gate.py:63-66`). T05 does not specify how an `ABANDON` verdict should map to a valid gate verdict.
