## Objectives & Success Criteria

`find_repo_root()` requires `.git` in ancestry and dies with a clear error instead of silent cwd fallback. `list_features` accepts any digit-width directory prefix (not just 3-digit). `find_feature_dir` supports `--auto` resolution (most recently modified feature). Slug sanitization collapses double hyphens and strips trailing hyphens.

## Subtasks

1. Rewrite `find_repo_root()`: walk up from cwd, require both `.git` and `design/specs/` in the same directory (or `.git` in a parent). For `init`, require `.git` before creating structure. On failure, `die("No git repository with design/specs/ found between cwd and filesystem root")`
2. Change `list_features` regex from `r"^\d{3}-"` to `r"^\d+-"` in the `re.match` call
3. Add `--auto` flag to argparse for all commands accepting `<feature>`. Implementation: find the most recently modified feature directory via `max(list_features(root), key=lambda d: d.stat().st_mtime)`
4. Update slug sanitization in `cmd_init`: add `slug = re.sub(r"-+", "-", slug).strip("-")` after existing transforms
5. Verify `find_feature_dir` resolves `7`, `007`, `007-spec`, and `007-spec-helper-v2` all to the same directory

## Test Strategy

- `test_find_repo_root_with_git_and_specs` — returns correct root when both exist
- `test_find_repo_root_no_git_dies` — dies when no `.git` in ancestry (use tmp dir)
- `test_find_repo_root_no_specs_dies` — dies when `.git` exists but no `design/specs/`
