## Objectives & Success Criteria

`init` creates only the feature directory (no `spec.md` template, no `tasks/` subdirectory). Error output in JSON mode returns `{"error": "...", "code": "..."}` to stdout with exit code 1. The `--json` flag is defined only at the global parser level (no duplicate `json_sub` on subcommands). `status` warns on stderr for invalid lane values instead of silently bucketing as "planned".

## Subtasks

1. Rewrite `cmd_init`: remove `SPEC_TEMPLATE` write and `tasks/` directory creation. Only create `design/specs/NNN-slug/` and return path. Human output: `"Created: design/specs/007-user-auth/"`. JSON output: `{"feature_number": "007", "slug": "...", "feature_dir": "..."}`
2. Implement structured error output: replace `die()` with a function that checks the `--json` flag. In JSON mode: `print(json.dumps({"error": msg, "code": error_type}))` to stdout + `sys.exit(1)`. In human mode: `print(f"error: {msg}", file=sys.stderr)` + `sys.exit(1)`
3. Remove `--json` from all subcommand parsers. Keep only the global `--json` flag. Remove `json_sub` merge logic from `main()`
4. In `cmd_status`, when a WP has an unrecognized lane value: print `f"warning: {filename} has unknown lane '{lane}', treating as planned"` to stderr, then bucket as planned (current fallback behavior, now with warning)
