## Objectives & Success Criteria

The hand-rolled YAML parser (`parse_frontmatter`, `update_frontmatter_field`, lines 207-255) is replaced with `python-frontmatter`. The stdlib `validate_wp_metadata` and `normalize_wp_metadata` functions are implemented. All existing WP files across features 001-006 can be read and normalized without errors. Unknown fields (e.g., `issue` in 006 WPs) are preserved through read-write cycles.

## Subtasks

1. Add `import frontmatter` with a graceful ImportError that prints `"spec-helper requires python-frontmatter: pip install python-frontmatter"` and exits 1
2. Implement `validate_wp_metadata(meta, filename) -> list[str]` as specified in design.md ‚Äî validates `work_package_id`, `title`, `lane`, `depends_on`
3. Implement `normalize_wp_metadata(raw, filename) -> dict` as specified in design.md ‚Äî handles `depends` -> `depends_on` (string, list, and empty), missing `work_package_id` derived from filename
4. Replace `parse_frontmatter()` and `update_frontmatter_field()` with `frontmatter.load()` / `frontmatter.dump()`
5. Remove `SPEC_TEMPLATE` and `WP_TEMPLATE` constants (dead code ‚Äî neither is used by any caller)
6. Verify round-trip: load a 006 WP file with `issue` field, modify `lane`, dump ‚Äî `issue` field must survive
