## Objectives & Success Criteria

Overhaul the consolidation nudge to be less aggressive. After this WP:

- Default threshold is 10 (up from 5), consistent across all three definition sites
- Two-marker system: `.last-nudge-<project_key>` (per-project cooldown) + `.last-nudge-global` (cross-project accumulation guard)
- 24-hour cooldown after each nudge — no re-nudge within 24 hours per-project or globally
- 30% randomized firing probability when threshold is met and cooldowns have expired
- Users who have never consolidated and meet the threshold get the probability gate without pre-written suppression marker
- `consolidation_check.py` uses `get_db_connection()` instead of raw `sqlite3.connect()`
- `write_config.py` imports default from `db.py` instead of maintaining its own copy

## Subtasks

1. Update `db.py:DEFAULT_SETTINGS` — change `consolidation_min_sessions` from 5 to 10.

2. Update `hooks/write_config.py` — remove the local `DEFAULT_CONFIG` dict. Import `DEFAULT_SETTINGS` from `db.py` and derive config defaults from it. The `main()` function starts from `DEFAULT_SETTINGS.copy()` instead of `DEFAULT_CONFIG.copy()`.

3. Refactor `hooks/consolidation_check.py`:
   - Switch from raw `sqlite3.connect()` to `get_db_connection()`.
   - Replace hardcoded fallback `settings.get("consolidation_min_sessions", 5)` with `settings.get("consolidation_min_sessions", DEFAULT_SETTINGS["consolidation_min_sessions"])`.
   - Add `NUDGE_PROBABILITY = 0.30` as a named module constant.
   - Add `read_last_nudge(project_key) -> datetime | None` — reads `~/.claude-memory/.last-nudge-<project_key>`.
   - Add `read_global_nudge() -> datetime | None` — reads `~/.claude-memory/.last-nudge-global`.
   - Add `write_nudge_markers(project_key)` — writes both per-project and global markers with current ISO timestamp.
   - Update `main()`:
     a. After `should_nudge = True`, check per-project cooldown (24h since last nudge) and global cooldown (24h since global nudge). If either is within 24h, suppress.
     b. Apply probability gate: `if random.random() >= NUDGE_PROBABILITY: suppress`.
     c. If nudge fires, call `write_nudge_markers(project_key)` before printing the JSON output.
     d. For never-consolidated users (last_ts is None): apply probability gate but do NOT pre-write markers on first upgrade — let the nudge fire naturally.

## Test Strategy

- Existing consolidation tests must pass (may need updating for new default)
- New tests in `tests/test_consolidation.py` (new file, or extend existing if test coverage exists in another file):
  - `test_default_threshold_is_10` — verify DEFAULT_SETTINGS has 10
  - `test_cooldown_suppresses_within_24h` — write a nudge marker 1 hour ago, verify no nudge fires
  - `test_cooldown_allows_after_24h` — write a nudge marker 25 hours ago, verify nudge can fire
  - `test_global_cooldown_suppresses_cross_project` — write global marker 1 hour ago for project A, verify project B is also suppressed
  - `test_randomization_with_seed` — seed `random.random`, verify nudge fires on some calls and not others at 30% rate
  - `test_never_consolidated_no_presuppression` — no markers exist, 15+ sessions, verify probability gate applies without writing a suppression marker first
  - `test_write_config_imports_from_db` — verify write_config uses DEFAULT_SETTINGS, not its own dict
  - `test_uses_get_db_connection` — verify consolidation_check does not call raw sqlite3.connect
