## Objectives & Success Criteria

Split the 2600-line `ingest_token_data.py` into four modules, each under 800 lines. After this WP:

- `token_schema.py` — schema definitions, ensure_schema(), version management
- `token_parser.py` — JSONL parsing, data classes, session parsing
- `token_analytics.py` — query functions, insight generation, cost calculation, trend analysis
- `token_dashboard.py` — dashboard deployment, main() entry point
- `pyproject.toml` entry point updated to `claude_memory.token_dashboard:main`
- All four modules use `get_db_path()` from `db.py` instead of local `DB_PATH` constant
- `DASHBOARD_OUT_PATH` is computed inside `main()` (not at module level) to respect custom db_path
- All existing token ingest tests pass

## Subtasks

1. Create `src/claude_memory/token_schema.py` — move `SCHEMA_SQL`, `SCHEMA_VERSION`, `ensure_schema()`, and the token-specific table definitions (turns, turn_tool_calls, session_metrics, hook_executions, token_import_log). Import `get_db_path` from `db.py` instead of defining `DB_PATH`.

2. Create `src/claude_memory/token_parser.py` — move data classes (`JnlFile`, `ToolCall`, `Turn`, `ParsedSession`), parsing functions (`parse_session`, `_parse_timestamp`, `_extract_usage`, `_detect_cache_ttl_ms`, `compute_session_analytics`), discovery functions (`discover_jsonl_files`, `should_skip_file`, `record_import`), helper functions (`_decode_project_cwd`, `_project_slug`), and constants (`BATCH_SIZE`, `PROGRESS_INTERVAL`, `COMMAND_TRUNCATE`, `_BASH_ANTIPATTERN_PREDICATE`, `MODEL_PRICING`, `_get_pricing`, `_turn_cost`).

3. Create `src/claude_memory/token_analytics.py` — move `build_output()`, `build_trends()`, `_build_insights()`, `_insights_to_findings()`, `_insights_to_recommendations()`, `import_session()`, `backfill_token_snapshots()`. These are the query/analysis functions that read from the database after ingestion.

4. Create `src/claude_memory/token_dashboard.py` — move `deploy_dashboard()` and `main()`. The `main()` function orchestrates: connect to DB → ensure schema → discover files → parse/import → backfill → build output → deploy dashboard → print slim JSON. Compute `DASHBOARD_OUT_PATH` inside `main()` from `get_db_path().parent / "dashboard.html"` (not at module level). Import from the three sibling modules.

5. Delete `src/claude_memory/ingest_token_data.py` after all four modules are created and tested.

6. Update `pyproject.toml` — change `cm-ingest-token-data` entry point from `claude_memory.ingest_token_data:main` to `claude_memory.token_dashboard:main`.

7. Update `__init__.py` docstring to list new token submodules.
