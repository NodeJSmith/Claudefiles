## Objectives & Success Criteria

Improve `detect_disposition()` to use `commits` data for COMPLETED detection, rename INTERRUPTED‚ÜíABANDONED, and enrich aggregated_content with file paths and commits for FTS searchability. After this WP:

- `detect_disposition()` accepts an optional `commits` parameter; non-empty commits ‚Üí COMPLETED
- The INTERRUPTED return value is renamed to ABANDONED everywhere
- `aggregated_content` includes full file paths and commit text (SET, not append)
- FTS search for a filename returns sessions where that file was edited
- The summary backfill target version is bumped from 2 to 3
- All callers of `detect_disposition()` are updated

## Subtasks

1. Update `summarizer.py:detect_disposition()` ‚Äî add `commits: list[str] | None = None` parameter. When `commits` is non-empty, return `COMPLETED`. Rename the `INTERRUPTED` return on line 51 to `ABANDONED`. Keep existing text-based heuristics as the fallback.

2. Update `summarizer.py:build_context_summary_json()` ‚Äî pass the `commits` list (parsed from `branch_row["commits"]`) to `detect_disposition()`.

3. Update `summarizer.py:build_context_summary_json()` ‚Äî apply `truncate_mid()` to exchange text in the JSON output (user and assistant fields in first_exchanges and last_exchanges). This bounds the JSON size.

4. Update `session_ops.py` (from WP01) ‚Äî when computing `aggregated_content`, SET it to: message text concatenation + `\n__files__\n` + deduplicated full file paths (one per line) + `\n__commits__\n` + commit text (one per line). The `__files__` and `__commits__` markers are for human readability of the stored blob, not for parsing.

5. Update `hooks/memory_context.py:_build_fallback_context()` ‚Äî refactor to construct a summary_json dict from the raw session data and call `render_context_summary()` from summarizer.py. Remove the ~120 lines of duplicated rendering logic. Pass `commits` from the session entry to `detect_disposition()`.

6. Update `hooks/backfill_summaries.py` ‚Äî change the query from `summary_version < 2` to `summary_version < 3`. This ensures the improved disposition heuristics propagate to all existing branches via the background backfill process.

7. Update `hooks/memory_setup.py:_needs_backfill()` ‚Äî change the check from `summary_version < 2` to `summary_version < 3`.
