## Objectives & Success Criteria

Fix operational issues in hooks: background process races, temp file leaks, and raw sqlite3 connections. After this WP:

- `_spawn_background()` uses an atomic PID-file guard — at most 1 concurrent import and 1 concurrent backfill
- `memory_sync.py` temp file is cleaned up on Popen failure
- `memory_setup.py` reaps stale `claude-memory-sync-*.json` files on SessionStart
- `memory_setup.py` helper functions use `get_db_connection()` instead of raw `sqlite3.connect()`
- `recent_chats.py` and `search_conversations.py` use `get_db_connection()` instead of raw `sqlite3.connect()`

## Subtasks

1. Add PID-file guard to `hooks/memory_setup.py:_spawn_background()`:
   - Before spawning, attempt `os.open(pid_path, os.O_CREAT | os.O_EXCL | os.O_WRONLY)` for the process-specific PID file (e.g., `~/.claude-memory/.pid-cm-import-conversations`).
   - If exclusive create succeeds: spawn the process, write the child PID to the file, close the fd. The spawned process must delete its own PID file on exit.
   - If exclusive create fails (FileExistsError): read the existing PID, check liveness via `os.kill(pid, 0)`. If dead, unlink the stale file and retry. If alive, skip spawning.

2. Update the spawned processes (`import_conversations.py:main()` and `backfill_summaries.py:main()`) to delete their own PID file on exit (in a `finally` block). The PID file path must be passed as an argument or derived from a convention (`~/.claude-memory/.pid-<entry-point-name>`).

3. Fix `hooks/memory_sync.py` temp file leak — wrap the `subprocess.Popen` call in a try/finally that unlinks `tmp_path` if Popen fails. Only skip cleanup when Popen succeeds (cm-sync-current handles its own cleanup via `--input-file` read + unlink).

4. Add stale temp file reaper to `hooks/memory_setup.py:main()` — on SessionStart, scan `tempfile.gettempdir()` for `claude-memory-sync-*.json` files older than 1 hour and delete them.

5. Update `hooks/memory_setup.py:_needs_reimport()` and `_needs_backfill()` to use `get_db_connection()` instead of raw `sqlite3.connect()`. This ensures WAL mode, busy_timeout, and migrations are applied.

6. Update `recent_chats.py:main()` — replace `sqlite3.connect(args.db)` with `get_db_connection()`. Remove manual WAL/busy_timeout pragmas (get_db_connection handles them). Ensure the `--db` CLI flag still works by passing the user-specified path to `get_db_connection({"db_path": str(args.db)})` when `args.db` differs from the default.

7. Update `search_conversations.py:main()` — same change as subtask 6, preserving `--db` flag override.

## Review Guidance

- **Critical**: PID guard must use O_CREAT | O_EXCL (atomic) — not check-then-create
- Verify spawned processes delete their PID file in a `finally` block, not just on success
- Verify the stale temp reaper only deletes files matching the `claude-memory-sync-*` pattern, not all temp files
- Verify the reaper uses file mtime for age check, not creation time
- Check that get_db_connection replacement doesn't break the `--db` CLI flag in recent_chats/search_conversations (the flag overrides the default path)
