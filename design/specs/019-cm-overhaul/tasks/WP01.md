## Objectives & Success Criteria

Extract the duplicated session import logic from `sync_current.py` (468 lines) and `import_conversations.py` (636 lines) into two new shared modules. After this WP:

- `session_ops.py` contains the single session import function used by both callers
- `project_ops.py` contains the shared project upsert logic
- Both `sync_current.py` and `import_conversations.py` are thin callers delegating to the shared modules
- Neither file contains duplicated SQL for session/message/branch operations
- The sync path writes `import_log` with `file_hash = NULL` for dedup consistency
- `has_tool_use` and `tool_summary` are no longer written in INSERT statements
- All existing tests pass without modification (refactor, not behavior change)

## Subtasks

1. Create `src/claude_memory/project_ops.py` — extract project key normalization, path derivation from cwd, and the INSERT/UPDATE/SELECT cascade from both files. Function signature: `upsert_project(cursor, project_key, cwd=None, project_dir=None) -> int` where `cwd` is used directly when available (sync path) and `project_dir` triggers JSONL-probe strategy when `cwd` is absent (import path).

2. Create `src/claude_memory/session_ops.py` — extract the shared logic: session upsert, message insertion with UUID dedup (without `has_tool_use`/`tool_summary` in the INSERT), branch detection via `find_all_branches`, branch metadata computation, branch_messages diff (add/remove), aggregated content assembly via a shared `build_aggregated_content(message_texts, file_paths, commits) -> str` helper (also used by the v5 migration in WP03), context summary computation. Function signature: `sync_session(conn, filepath, project_dir, write_import_log=False, file_hash=None) -> int`. When `write_import_log=True` and `file_hash` is provided, write to import_log. When called from sync, pass `file_hash=None` to write a NULL-hash entry.

3. Refactor `hooks/sync_current.py` to: read hook input, find session file, call `project_ops.upsert_project(cursor, project_key, cwd=meta["cwd"])`, then `session_ops.sync_session(conn, filepath, project_dir, write_import_log=True, file_hash=None)`. Remove all duplicated SQL.

4. Refactor `hooks/import_conversations.py` to: scan directory, compute file hash, check import_log for skip, call `project_ops.upsert_project(cursor, project_key, project_dir=project_dir)`, then `session_ops.sync_session(conn, filepath, project_dir, write_import_log=True, file_hash=hash)`. Remove all duplicated SQL. The `import_project()` function stays as the directory-scanning orchestrator.

5. Confirm whether `parsing.py:aggregate_branch_content()` is still needed or fully superseded by `session_ops.py`. If superseded, remove it. If still called elsewhere (e.g., by the v5 migration path), keep it but note the relationship.

6. Update `__init__.py` docstring to list new submodules.
