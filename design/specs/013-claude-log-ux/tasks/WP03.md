## Objectives & Success Criteria

- `claude-log search "pattern"` returns conversation turns: each match shows the matched entry plus the preceding entry (user prompt before assistant response, or vice versa)
- Results are grouped by session with a header line per session (`--- <id> | <project> | <date> ---`)
- Sessions are searched in recency order (most recent first) via mtime-sorted `iter_session_files`
- Matched entries truncated at 500 chars; preceding entries also truncated at 500 chars
- Subagent matches include agent label and use `last_user_entry` as preceding context
- `--limit` applies to total results across all sessions (default: 50); stderr notice when limit is hit
- Exit code 1 when no matches found; exit code 0 with matches
- All existing flags preserved: `--type`, `--since`, `-F`, `--fixed`, `--limit`, `-p`, `--project`
- `--json` produces updated JSON output with `matched` and `preceding` fields (longer text, grouped by session)

## Subtasks

1. Modify `iter_session_files()` to accept an optional `sort_by_mtime: bool = False` parameter; when True, collect all paths, stat each for mtime, sort descending, then yield
2. Update `cmd_search` to pass `sort_by_mtime=True` to `iter_session_files`
3. Restructure `cmd_search` search loop to track `last_user_entry` and `last_assistant_entry` per session
4. For each match from `_match_entry`, capture `{"matched": entry, "preceding": <appropriate entry>, "session_id": ..., "project": ..., "timestamp": ...}`
5. For subagent (progress entry) matches, use `last_user_entry` as preceding; guard formatters against `preceding: None`
6. Group results by session ID before output
7. Add stderr notice when `--limit` is reached: `"Showing first N results — use --limit to increase or --since to narrow."`
8. Add exit code 1 for no matches (currently returns None → exit 0)
9. Create `_format_search(grouped_results: list[dict]) -> str` — session headers, conversation turn pairs, 500-char truncation
10. Wire through `_output()` dispatch
11. Update JSON output to include `preceding` field and longer `text` (500 chars instead of 120)

## Test Strategy

- `class TestSearchConversationTurns`: test that results include preceding entry; test assistant match has user preceding; test user match has assistant preceding; test subagent match has user preceding; test `preceding: None` when no prior entry
- `class TestSearchGrouping`: test results are grouped by session with headers in text output
- `class TestSearchRecencySort`: test that sessions are searched most-recent-first (mock `iter_session_files` with mtime sorting)
- `class TestSearchLimit`: test `--limit` stops at N total results; test stderr notice is emitted
- `class TestSearchExitCodes`: test exit 1 on no matches; test exit 0 on matches
- `class TestFormatSearch`: test session header format; test truncation at 500 chars; test multi-session grouping
- `class TestSearchFlagsPreserved`: test `--type tool_use` still works; test `--since`, `-F`, `--project` still work
