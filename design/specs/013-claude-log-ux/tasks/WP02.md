## Objectives & Success Criteria

- `claude-log show <id>` (no flags) outputs orientation mode: session metadata, first user message, `~N entries` count, last 3 entries
- First user message skips system-injected entries (XML tags, tool_results, /commands) using structural heuristic
- Ring buffer entries are trimmed to 10KB max to prevent memory bloat
- Entry count uses `~` prefix (line-count estimate)
- `claude-log show <id> --grep <pattern>` searches full session, returns matched turns with preceding entry context
- `--grep` composes with type filters (`--tools`, `--messages`, etc.)
- `--grep` and `--tail` are mutually exclusive (exit with error)
- `show <id> --tools` absorbs `extract --tools` functionality
- `show <id> --messages` absorbs `extract --messages` functionality
- All filter flags (`--messages`, `--tools`, `--user`, `--assistant`, `--thinking`, `--usage`, `--all`, `--limit`, `--tail`) bypass orientation mode
- Exit code 1 when `--grep` finds no matches; exit code 0 with orientation mode or filtered output
- JSON output via `--json` preserves current `cmd_show` output structure

## Subtasks

1. Add `--grep`/`-g`, `--all`, `--tail` flags to `show` subcommand parser in `build_parser()`
2. Add mutual exclusivity check: error if both `--grep` and `--tail` are provided
3. Implement `_is_system_entry(entry: dict) -> bool` — returns True for entries with XML-prefixed content, tool_result types, or /command entries
4. Implement orientation mode in `cmd_show`: single-pass through `iter_entries()` capturing first non-system user message, running count, and last 3 entries via `collections.deque(maxlen=3)` with 10KB trim per entry
5. Create `_format_show_orientation(data: dict) -> str` — renders metadata header, first user message, omitted count, last entries
6. Create `_format_show_entries(turns: list[Turn]) -> str` — renders chronological conversation with `[user]`, `[assistant]`, `[Bash]` etc. prefixed lines, subagent entries indented
7. Implement `--grep` path: compile pattern, single-pass search through all entries (no limit), track `last_user_entry` and `last_assistant_entry`, collect matched turns with preceding context
8. Create `_format_show_grep(matched_turns: list[dict]) -> str` — renders matched sections separated by `---` dividers
9. Implement flag composition: `--grep` + `--tools` filters to tool-call entries before matching
10. Wire all paths through `_output()` dispatch
11. Add `--tools`, `--messages`, and `--usage` filter flags to `show` (these are NEW flags — they currently exist only on `extract`, not on `show`). Port the filtering logic from `cmd_extract`'s `_extract_tools()`, `_extract_messages()`, and `_extract_usage()` into `cmd_show`'s rendering pipeline
