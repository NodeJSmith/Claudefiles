## Objectives & Success Criteria

A new `bin/lint-cli-conventions` script exists that:
1. Checks every executable script in `bin/` for `--help` handling (bash: conditional/case pattern; Python: argparse import). Plumbing scripts in an inline exemption set are skipped.
2. Parses the CLI Tools table in `capabilities.md` and verifies each listed tool name corresponds to an executable in `bin/`. Tools marked with `# EXTERNAL: tool-name` comments are exempt.
3. Exits 0 if all checks pass, exits 1 with clear messages if any fail.

The script is wired as a `repo: local` pre-commit hook with `always_run: true`.

## Subtasks

1. Create `bin/lint-cli-conventions` as a bash script. Implement two checks:
   - **`--help` presence**: iterate over executable files in `bin/` (skip `ado-common.sh` and files in an exemption list). For `.sh`/no-extension files, grep for `--help` in a case/conditional pattern. For `.py` files, grep for `import argparse`. Report failures.
   - **`capabilities.md` sync**: extract tool names from the CLI Tools table (the `| Run |` column). For each, check `bin/<tool>` exists. Parse `# EXTERNAL:` comments for exemptions. Report mismatches.
2. Add the script's own `--help`/`-h` handler (it must pass its own lint check).
3. Add the `repo: local` hook stanza to `.pre-commit-config.yaml`:
   ```yaml
   - repo: local
     hooks:
       - id: lint-cli-conventions
         name: CLI convention lint
         entry: bin/lint-cli-conventions
         language: system
         pass_filenames: false
         always_run: true
   ```
4. Run the lint check to verify it passes against the current state (after WP03-WP05 have added `--help` to all scripts).

## Test Strategy

N/A — no test infrastructure in this repo. Manual verification:
- Run `bin/lint-cli-conventions` directly — verify exit 0 (all checks pass)
- Temporarily remove `--help` from one bin/ script — verify lint check catches it and exits 1
- Add a fake tool name to `capabilities.md` — verify lint check catches the missing executable
- Add a `# EXTERNAL: fake-tool` comment — verify lint check exempts it
- Run `bin/lint-cli-conventions --help` — verify it shows usage
- Run `pre-commit run lint-cli-conventions` — verify the hook fires

## Review Guidance

- **Spec reviewer**: verify both checks are implemented (--help presence AND capabilities.md sync). Verify the exemption mechanisms work (inline list for plumbing scripts, `# EXTERNAL:` comments for capabilities.md).
- **Code reviewer**: verify the `--help` grep pattern checks for case/conditional use, not just any mention of `--help` in the file (comments, usage strings would false-positive). Verify the capabilities.md parser correctly extracts tool names from the markdown table.
- **Integration reviewer**: verify the pre-commit hook stanza is correct (`language: system`, `pass_filenames: false`, `always_run: true`). Verify it doesn't conflict with existing hooks.
