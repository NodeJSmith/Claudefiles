## Objectives & Success Criteria

All subcommand-style bash scripts respond to `--help` with usage text, examples, and exit code 0:
- `ado-builds --help` exits 0 (currently exits 1 via `usage()`)
- `ado-logs --help` exits 0
- `ado-pr --help` exits 0
- `ado-pr-threads --help` exits 0
- `claude-tmux --help` shows help even outside tmux (currently prints "Not in tmux")

Each tool also supports `<tool> <subcommand> --help` with per-subcommand examples.

Additionally: `bin/ado-pr-threads` gets a comment explaining the API version 7.0 pin.

## Subtasks

1. For each ADO script (`ado-builds`, `ado-logs`, `ado-pr`, `ado-pr-threads`): add `--help|-h)` as the **first case** in the top-level dispatch. Print usage text with 2-3 examples and call `exit 0`. Do NOT modify `usage()` — it stays at `exit 1` for error cases.
2. For each ADO script: add `--help` handling within each subcommand's argument parsing (so `ado-builds list --help` shows list-specific help).
3. Add examples to the help output of each ADO script showing real-world invocations with common flags (e.g., `ado-builds list --json`, `ado-pr show --json`, `ado-pr-threads list --json --all`).
4. For `bin/claude-tmux`: add the `--help`/`-h` check **before** the TMUX guard at line 19. This ensures `claude-tmux --help` works outside tmux. Include examples for each subcommand (rename, current, new, list, panes, capture, kill).
5. Add a comment to `bin/ado-pr-threads` at the `ADO_API_VERSION="7.0"` line explaining the version pin: `# Pin to 7.0 — thread API behavior differs in 7.1; not yet verified compatible`.
