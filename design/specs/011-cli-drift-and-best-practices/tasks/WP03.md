## Objectives & Success Criteria

All simple bash scripts in `bin/` respond to `--help` and `-h` with usage text, 2-3 examples, and exit code 0:
- `git-branch-base`
- `git-branch-diff-files`
- `git-branch-diff-stat`
- `git-branch-log`
- `git-default-branch`
- `git-platform`
- `get-skill-tmpdir`
- `get-tmp-filename`

Running `<script> --help` no longer silently executes or produces garbage output (e.g., `get-skill-tmpdir --help` no longer creates a temp dir named `claude---help-*`).

## Subtasks

1. Add `--help`/`-h` guard to `bin/git-branch-base` before any logic. Include usage and 2-3 examples showing common invocations.
2. Add `--help`/`-h` guard to `bin/git-branch-diff-files`. Include examples.
3. Add `--help`/`-h` guard to `bin/git-branch-diff-stat`. Include examples.
4. Add `--help`/`-h` guard to `bin/git-branch-log`. Include examples.
5. Add `--help`/`-h` guard to `bin/git-default-branch`. Include examples.
6. Add `--help`/`-h` guard to `bin/git-platform`. Include examples.
7. Add `--help`/`-h` guard to `bin/get-skill-tmpdir`. Include examples (note the "do NOT wrap in `$()`" caveat).
8. Add `--help`/`-h` guard to `bin/get-tmp-filename`. Include examples.

Pattern for each:
```bash
[[ "${1:-}" == "--help" || "${1:-}" == "-h" ]] && {
    cat <<'EOF'
Usage: <script> [args]

<One-line description>

Examples:
  <script> <example1>    # <comment>
  <script> <example2>    # <comment>
EOF
    exit 0
}
```

- **Integration reviewer**: spot-check that the help text accurately describes what each script does (compare to the script body).
