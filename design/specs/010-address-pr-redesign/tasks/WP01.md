## Objectives & Success Criteria

A new `bin/git-platform` script exists that:
- Reads `git remote get-url origin` and prints `github`, `ado`, or `unknown` to stdout
- Always exits 0 (never non-zero — LLM error handling is confused by non-zero exits)
- Works from any directory within a git repo
- Is executable and follows existing `bin/` conventions (shebang, `set -euo pipefail`, comment header)
- Is symlinked into `~/.local/bin/` via `install.sh`

## Subtasks

1. Create `bin/git-platform` (~15 lines):
   - `#!/usr/bin/env bash` + `set -euo pipefail`
   - `git remote get-url origin 2>/dev/null` — capture the remote URL
   - If URL contains `github.com` → print `github`
   - If URL contains `dev.azure.com` or `visualstudio.com` → print `ado`
   - Otherwise → print `unknown`
   - Always exit 0
2. Make the script executable: `chmod +x bin/git-platform`
3. Verify `install.sh` handles new files in `bin/` correctly (it symlinks the entire directory — read `install.sh` to confirm no per-file registration is needed)

## Test Strategy

N/A — no test infrastructure in this repo. Validate by running `git-platform` in a GitHub repo and an ADO repo (if available) and confirming output.

## Review Guidance

- Verify the script uses `git remote get-url origin` (not `git config --get remote.origin.url` which has different quoting behavior)
- Verify exit code is always 0, even for `unknown`
- Verify string matching is case-insensitive or uses the exact casing GitHub/ADO use in remote URLs
- Verify the script doesn't use command substitution patterns that break under the Bash tool wrapper (per `CLAUDE.md` Bash Tool Restrictions)
