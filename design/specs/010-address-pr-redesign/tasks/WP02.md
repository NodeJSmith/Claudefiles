## Objectives & Success Criteria

The `bin/gh-pr-threads` script gains three capabilities:
- `--json` flag: emits the raw GraphQL response as JSON instead of human-readable text
- `--all` flag: includes resolved threads (currently filtered out by the `--jq` post-processing)
- Pagination: handles >100 review threads via cursor-based pagination, returning all threads in a single output
- Existing behavior (no flags) is unchanged — human-readable output, unresolved only
- The `author` selection set in the GraphQL query includes `__typename` for bot detection

## Subtasks

1. Read `bin/gh-pr-threads` fully to understand the current GraphQL query and `--jq` filtering
2. Add argument parsing at the top of the script:
   - `--json` flag (boolean)
   - `--all` flag (boolean)
   - PR number remains the first positional argument (optional, auto-detects if not given)
3. Add `__typename` to the GraphQL query's `author` fragment: change `author { login }` to `author { login __typename }`
4. Add `pageInfo { hasNextPage endCursor }` to the `reviewThreads` selection
5. Implement cursor-based pagination loop:
   - First call uses `reviewThreads(first: 100)`
   - If `hasNextPage` is true, loop with `reviewThreads(first: 100, after: "<endCursor>")`
   - Accumulate all `nodes` arrays into a single result
   - Stop when `hasNextPage` is false
6. When `--json` is passed:
   - Skip the `--jq` formatting filter
   - Output the accumulated thread nodes as a JSON array to stdout
   - If `--all` is NOT passed, filter to `isResolved: false` before output
7. When `--json` is NOT passed:
   - Keep existing `--jq` human-readable output
   - If `--all` is passed, remove the `select(.isResolved | not)` filter
8. When neither flag is passed: behavior is identical to current script
