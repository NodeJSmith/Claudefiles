## Objectives & Success Criteria

Phase 2 of mine.define begins with a premise challenge question ("What happens if we don't build this?") for moderate and complex features. The question is skipped when a brief.md from mine.grill already addresses cost of inaction. When the user reports low cost, a follow-up surfaces the signal and offers to continue, descope, or table.

## Subtasks

1. In `skills/mine.define/SKILL.md` Phase 2, insert a new "Premise check" subsection before the existing `### Always ask (all complexity levels)` heading. Add a heading `### Premise check (moderate+ only)` with:
   - Skip logic: check for brief.md existence; if the brief's "Risks and Concerns" or "Key Decisions Made" section addresses cost of inaction, skip the AskUserQuestion and note what was found. The premise check is NOT skippable on the mine.build Accelerated path.
   - AskUserQuestion block with `question: "Before we dig in — what happens if we don't build this? What's the cost of doing nothing?"`, `header: "Premise check"`, free-text input (no options)
   - Processing rules: low-cost answer → surface signal and offer continue/descope/table; real-pain answer → use to strengthen Problem section in Phase 4

2. Update the Phase 2 opening paragraph (the text block between `## Phase 2: Proportional Discovery` and the first `###` heading) to note that premise check fires first for moderate+ features before problem grounding
