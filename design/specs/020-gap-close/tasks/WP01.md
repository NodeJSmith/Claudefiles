Create `skills/mine.gap-close/REFERENCE.md` containing per-artifact-type checklists (design doc, brief, work package, general-purpose) with severity tags (Blocker/Should-address/Nice-to-have), answer-to-artifact conversion patterns with concrete Edit tool examples, and an example walkthrough.

## Subtasks

1. Create `skills/mine.gap-close/REFERENCE.md` with document header and structure.
2. Write the design doc checklist (~20 items) organized by section: Problem/Goals, User Scenarios, Functional Requirements, Edge Cases, Acceptance Criteria, Architecture, Failure Modes, Security/Access, Dependencies, Test Strategy, Open Questions. Each item has an ID (e.g., `DD-01`), severity class, gap question text, and target section in design.md.
3. Write the brief checklist (~12 items) covering: Idea clarity, Key Decisions captured, Scope Boundaries defined, Open Questions addressed, Risks identified, Codebase Context present.
4. Write the work package checklist (~10 items) covering: Deliverables specified, acceptance criteria per deliverable, dependencies on other WPs stated, test requirements present, scope bounded.
5. Write the general-purpose checklist (~8 items) covering: Structure (sections non-empty), clarity (no ambiguous statements), testability (claims verifiable), scope (boundaries stated), completeness (no TBD/TODO markers).
6. Write answer-to-artifact conversion rules section with concrete Edit tool examples (old_string/new_string) for each pattern: acceptance criteria → G/W/T, architecture/decision → prose paragraph, UI state → Trigger/Visual/Behavior/Exit spec, edge case → bullet point, default → verbatim in existing style.
7. Write an example walkthrough showing a design doc survey → triage → 2-3 gap questions → edits applied.
