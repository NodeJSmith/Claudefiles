## Objectives & Success Criteria

Create `skills/mine.gap-close/SKILL.md` with the 5-phase workflow: locate and classify artifact, survey against checklist, triage summary, conversational loop, sign-off. Must stay under ~100 lines per the write-skill quality checklist.

Success: SKILL.md exists with correct frontmatter (name, description with trigger phrases, user-invocable: true), 5 numbered phases, all user interaction via AskUserQuestion with 2-4 options, and line count under ~100. Passes all 8 items on the write-skill quality checklist.

## Subtasks

1. Write frontmatter with `name: mine.gap-close`, description starting with "Use when the user says:" and at least 3 trigger phrases ("close gaps in this design", "fill gaps in the spec", "lightweight design review", "completeness review"), and `user-invocable: true`.
2. Write Phase 1 (Locate and Classify): parse `$ARGUMENTS` (file path, feature directory, description, or empty), resolve to artifact file, read it, detect artifact type using the heuristic (design doc, brief, WP, general-purpose), load corresponding checklist from `${CLAUDE_HOME:-~/.claude}/skills/mine.gap-close/REFERENCE.md`. Include AskUserQuestion fallback when input is ambiguous. Reference `${CLAUDE_HOME:-~/.claude}/skills/mine.write-skill/REFERENCE.md` for the 8-item quality checklist used in subtask 7.
3. Write Phase 2 (Survey): instruct the LLM to evaluate each checklist item against artifact content, marking PASS or GAP with severity from checklist.
4. Write Phase 3 (Triage): present gap counts by severity and one-line descriptions. AskUserQuestion with options: "Walk through all gaps", "Blockers only", "Take the list and exit".
5. Write Phase 4 (Conversational Loop): iterate gaps in severity order. For each gap: before generating options, grep the codebase for existing patterns relevant to that gap's domain (e.g., error handling conventions, naming patterns) and surface the most common pattern as the first option with "(Recommended)". Present via AskUserQuestion with 2-3 contextually plausible answer options, receive answer, convert per REFERENCE.md rules, write into artifact via Edit, confirm with one-line ack. Include recap every 3-5 gaps via AskUserQuestion with "Continue", "Pause", "Skip remaining".
6. Write Phase 5 (Sign-Off): re-run checklist on updated artifact. If all blockers resolved: AskUserQuestion with "Approve", "Run full challenge" (invoke `/mine.challenge`), "Save and stop". If blockers remain: report count, offer "Continue" or "Save and stop".
7. Validate against the 8-item write-skill quality checklist. Fix any failures.
