Before doing anything, read these files completely:

1. Read the skill: `~/.claude/skills/mine.look-and-feel/SKILL.md`

---

## Execute the Skill

Follow the phases defined in SKILL.md exactly:

1. Check for existing `design/direction*.md` files
2. Gather intent (who, task, feel)
3. Collect visual references
4. Domain exploration
5. Self-review (swap, squint, signature, token tests)
6. Propose direction with concrete token values
7. Confirm and save to `design/direction.md` (or scoped variant)
8. Offer hand-off to `/mine.mockup`

## Communication

Be invisible. Don't announce modes or narrate process. Jump into work with reasoning. Every token choice must be justified against intent and domain.
